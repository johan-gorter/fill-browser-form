# Contributing to Browser Form Filler Skill

Thank you for your interest in contributing to this Claude Skill! Contributions are welcome and appreciated.

## How to Contribute

### Reporting Issues

If you encounter a bug or have a feature request:

1. Check if the issue already exists in the [Issues](https://github.com/johan-gorter/fill-browser-form/issues) section
2. If not, create a new issue with:
   - A clear, descriptive title
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Any relevant HTML structure or error messages

### Suggesting Enhancements

Have an idea to improve the skill? Open an issue with:
- A clear description of the enhancement
- Use cases where it would be helpful
- Examples if applicable

### Pull Requests

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes to `Skill.md` or other files
4. Test your changes with real-world forms
5. Commit your changes with clear commit messages
6. Push to your fork
7. Open a Pull Request with:
   - Description of changes
   - Why the changes are needed
   - Any testing you've done

## Development Guidelines

### Skill.md Structure

- Keep instructions clear and concise
- Maintain the existing workflow steps
- Test any new helper functions with real forms
- Document timing requirements clearly

### Code Quality

- JavaScript snippets should be:
  - Well-commented
  - Error-handling aware
  - Compatible with modern browsers

### Testing

Before submitting changes, test with:
- Different types of forms (simple, complex, dynamic)
- Various browsers (Chrome, Firefox, Edge)
- Different field types (text, select, radio, checkbox)

## Questions?

Feel free to open an issue for any questions about contributing!

## Code of Conduct

Be respectful, constructive, and helpful. We're all here to make this skill better for everyone.
